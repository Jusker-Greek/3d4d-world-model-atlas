CHAPTER 01 · ONE-MINUTE MAP

先把三个最容易混淆的词拆开

蒸馏不是单一算法，而是一组“用教师信号训练学生”的方法。小模型也不一定来自蒸馏。

开放权重 ≠ 全流程开源

DeepSeek-R1-Distill 可下载且许可宽松，但原始 80 万条蒸馏数据与完整训练流水线没有全部公开；Open-R1 才补上了较完整的模型、数据、训练与评估链。

蒸馏 ≠ 只学最终答案

学生可以学教师的 token 概率、生成回答、推理轨迹、偏好排序、中间表示、多个教师的互补知识，甚至剪枝后如何恢复性能。

小模型 ≠ 蒸馏模型

TinyLlama、SmolLM、MobileLLM-R1 等可以主要靠架构、数据和训练配方得到。判断依据应是技术报告是否明确使用教师信号，而不是名字里有没有 “mini”。

本 survey 的口径：把“开源”拆成权重 / 代码 / 蒸馏数据三层；把“conditional”视为条件化或自适应蒸馏的工程总称，而不是一个已经统一定义的学术门类。
CHAPTER 02 · THREE AXES

用三根轴，定位任何一篇蒸馏工作

AXIS A · TRANSFER

传什么？

  • 分布：logits / token 概率
  • 行为：回答、工具轨迹、CoT
  • 结构：hidden states、关系、表征
  • 偏好：排序、critique、reward
AXIS B · ACCESS

能看到什么？

  • 白盒：权重、logits、中间层
  • 灰盒：top-k 概率或批量推理
  • 黑盒：只能拿文本输出
  • 多教师：不同架构 / tokenizer
AXIS C · CONDITION

目标何时变化？

  • 固定：离线教师数据
  • on-policy：跟着学生生成分布变
  • 课程式：随训练时间改变教师强度
  • 样本式：按难度、验证结果或预算路由
L = λ · L_ground_truth + α(t, x) · D(student || teacher / assistant) + β · L_preference_or_verifier

这条式子最重要的不是符号，而是三个选择：监督来源、分布距离、以及系数是否随训练步或样本变化。

CHAPTER 03 · FIVE FAMILIES

五类方法，比“普通 / 特殊 / conditional”更可操作

P(y|x)

白盒教师最典型的“普通蒸馏”。

Token / logit distribution matching

让学生在每个 token 上接近教师分布。MiniLLM 用 reverse KL 抑制低概率区域被过度估计；DistiLLM 用 skewed KL 与自适应离线策略；CSD（ICLR 2026）直接对齐相对 logit 差，绕开 softmax 平滑。

Response / rationale / trajectory distillation

教师先生成回答或思维链，再用 SFT、偏好学习或 RL 训练学生。DeepSeek-R1-Distill、OpenR1、Phi-4-mini-reasoning 都属于这条主线。关键不在“生成更多”，而在可验证、去污染、去重复和难度配比。

Domain / skill / preference distillation

把数学、代码、医疗、检索、embedding、工具使用或对齐偏好做成专项学生。专项模型常能在窄任务超过教师，但边界外能力会明显掉。HuatuoGPT-o1 和 EmbeddingGemma 是两个不同例子：前者蒸馏医疗推理数据，后者蒸馏几何 embedding 关系。

Conditional / adaptive / on-policy distillation

GKD 用学生自己生成的序列暴露错误；TAID 随训练时间把目标从学生分布逐步移向教师；AMiD 用 α-mixture assistant distribution 调整 mode-seeking 与 mode-covering；FutureMind 按问题阶段注入不同推理先验。

Prune-then-distill / cross-tokenizer / knowledge fusion

Minitron 与 Llama 3.2 先从大模型剪出小结构，再用 KD 恢复；FuseLLM 把多个教师的互补分布融合到一个学生；2026 的 Cross-Tokenizer Likelihood Scoring 则让不同 BPE 词表也能做概率级蒸馏。

CHAPTER 07 · TRAINING RECIPES

四条最小可行路线

ROUTE A · BLACK-BOX

只有教师输出：先做可验证的 response / CoT 蒸馏

  1. 从真实请求或公开题库抽样，先留独立测试集。
  2. 让教师多次采样，保留答案、必要推理和元数据。
  3. 用执行器 / 规则 / 检索证据验证；去重、去污染、控长度。
  4. 先 SFT，再用正确/错误轨迹做偏好优化或 verifier-RL。
适合：DeepSeek-R1 → Qwen / Llama 学生，数学、代码、医疗等可验证任务。
不要：直接把一条长 CoT 当真值；不要用教师自己给自己打分作为唯一过滤。
ROUTE B · WHITE-BOX

同 tokenizer 且能取 logits：GKD / DistiLLM 起步

  1. 用 SFT 或学生初始化建立可用起点。
  2. 混合真实数据与学生 on-policy 生成，暴露学生会犯的错。
  3. 教师在这些 token 前缀上给分布；先试 skew KL / GKD。
  4. capacity gap 大时加入 TAID 或 AMiD 的 assistant distribution。
适合：同模型家族压缩、可同时部署教师和学生的训练环境。
不要：只在教师序列上训练却期待学生自由生成时同样稳定。
ROUTE C · STRUCTURAL

已有大模型：剪枝 → 轻量恢复 → 蒸馏

  1. 确定目标硬件约束，再选宽度、深度、头数或 FFN 剪枝。
  2. 结构化剪枝并做少量 teacher correction。
  3. 用原教师分布继续预训练恢复，再做指令后训练。
  4. 量化是最后一层部署优化，不要把它和 KD 混为一谈。
适合：Minitron / Llama 3.2 路线，要保持 tokenizer、生态和模型家族一致。
不要：先随意缩结构，再用少量 SFT 期待恢复全部基础能力。
ROUTE D · CROSS-FAMILY

不同 tokenizer 或多教师：先对齐概率空间

  1. 若只取文本，直接做字符串级 response distillation。
  2. 若要概率级 KD，使用 ALM / contextual mapping / 2026 BPE scoring。
  3. 多个教师先做 task-wise 评估，避免弱教师在强项外拖累。
  4. 用 FuseLLM 式融合或路由，保留教师来源与样本标签。
适合：Qwen ↔ Llama ↔ Gemma 跨家族迁移、端侧缩词表、多教师集成。
不要：把不同 tokenizer 的 token id 直接做 KL 对齐。
CHAPTER 08 · 2026 FRONTIER

最新进展：不再只问“用哪个 KL”

2023–24
生成式 KD 的目标函数与 on-policy 问题被系统化

MiniLLM、GKD、DistiLLM 把焦点从分类式 KD 转向自回归生成、学生 rollout 与曝光偏差。

2024
蒸馏进入公开基础模型生产线

Llama 3.2、Minitron、Gemma 2/3 展示剪枝 + KD、预训练 logits 与指令教师如何进入模型家族构建。

2025
推理轨迹蒸馏成为主流

DeepSeek-R1-Distill、Open-R1、Phi-4-reasoning 把长 CoT、可验证过滤和后续 RL 组合成可复用范式。

ICLR 2026
assistant distribution 与跨 tokenizer 成熟

AMiD 系统化 α-mixture；Cross-Tokenizer Likelihood Scoring 用 BPE 递归结构做精确/近似概率转换；CSD 对齐相对 logits。

2026.06
Compress-Distill 研究“先压缩思维链再蒸馏”

用 Qwen3.5-397B-A17B 与 gpt-oss-120B 生成轨迹；压缩可把训练 token 降至原来的 12–30%，但原始轨迹仍保持最高准确率——这是速度/长度与精度的真实交换，不是免费午餐。

接下来
从答案模仿走向机制、交互和部署共同优化

值得追踪：circuit distillation、agent trajectory correction、diffusion LLM 轨迹蒸馏、以及按样本难度动态分配教师与思考预算。

CHAPTER 09 · EVALUATION & RISK

一个蒸馏项目真正失败，通常不是 loss 写错

① 能力任务准确率、生成质量、校准、长文本
② 效率首 token 延迟、吞吐、峰值显存、输出长度
③ 泛化新领域、难度外推、多语言、对抗提示
④ 合规教师许可、数据来源、PII、可追溯性
为什么学生可能学到“像在推理”，却没有真正更会推理？
文本轨迹监督首先复制的是行为分布。若数据不可验证、难度太低或风格高度重复，学生容易学会冗长模板。最小修正是：可执行验证、多样化问题、保留失败轨迹、同时评估最终答案与 token / 时间成本。
教师越大越好吗？
不一定。capacity gap 过大时，学生难以匹配尖锐或多峰的教师分布。Gemma 3 的消融显示，短训练期小教师可能更好，训练足够久大教师优势才反转；TAID / AMiD 正是在处理中间分布与训练稳定性。
为什么只用离线教师数据会失效？
训练时学生只看教师走过的正确前缀，推理时却会进入自己的错误前缀。GKD 和 DistiLLM 的 on-policy / adaptive off-policy 设计让教师在学生真实访问的状态上纠错。
“MIT / Apache”是不是就能放心商用？
仍需检查上游 base model、教师输出条款、训练数据和具体模型卡。例如 Qwen 派生模型通常承接 Apache 2.0，Llama 派生模型还受 Llama Community License 约束；Gemma 与 NVIDIA 也使用各自模型条款。页面标签只作工程筛选，不替代法律审查。
医疗等专业模型应该怎么用？
HuatuoGPT-o1 适合研究医疗推理和 RAG 原型，不应被当成临床诊断系统。专业蒸馏需要领域专家审核、来源可追溯检索、拒答机制和真实工作流评估，不能只看医学考试题。
